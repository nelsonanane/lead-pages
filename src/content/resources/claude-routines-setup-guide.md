## What's Inside

Claude Code Routines launched in April 2026 to 707 points on Hacker News. They're not local slash commands — they're cloud-hosted automation that runs Claude Code on Anthropic's infrastructure, autonomously, on any trigger you configure.

**You'll get:**
- A clear explanation of what Routines are (and what they're not)
- All three trigger types: Schedule (hourly/daily/weekly), API (HTTP POST endpoint per routine), and GitHub (fires on PRs, releases, pushes, etc.)
- Complete step-by-step setup from the web UI and CLI
- 5 concrete use cases mapped to trigger types: PR review, backlog triage, alert response, deploy verification, and docs drift detection

## Who This Is For

Developers and teams who use Claude Code and want to automate repetitive tasks that currently require someone to manually run them. If you've thought "I wish this ran automatically every night" or "I want Claude to review every PR without me triggering it" — Routines is exactly that.

Also for anyone who wants to wire Claude Code into existing pipelines: monitoring tools, CI/CD systems, deploy scripts — anything that can make an HTTP POST can now trigger a Claude session.

## The Core Unlock

Before Routines, Claude Code required a person to start every session. With Routines, Claude runs as a persistent cloud service. Your PR review checklist, your nightly triage, your post-deploy smoke test — these become autonomous processes that just happen, on schedule or on event, while you're doing other work.
