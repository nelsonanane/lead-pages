## What's Inside

Claude Code Routines launched in April 2026 to 700+ points on Hacker News. They let you define any multi-step AI workflow once, give it a name, and run it forever with one command — same steps, same output quality, every time.

**You'll get:**
- A plain-English explanation of what Routines are and when to use them
- Complete setup walkthrough from installing Claude Code to running your first Routine
- 5 starter Routines you can copy directly: `/review`, `/content-audit`, `/seo-check`, `/summarize`, `/refactor`
- Best practices: naming, versioning in git, sharing with teammates

## Who This Is For

Anyone who uses Claude Code and finds themselves re-typing the same prompt sequences. If you've run the same workflow more than twice — code review, document summary, content audit, SEO check — it should be a Routine.

Also for teams: Routines are stored as markdown files in your project. Commit them to git and everyone on the team runs the same workflow consistently.

## The Core Unlock

The value of Routines is consistency. Manual prompting drifts — you phrase things slightly differently each time, get slightly different output quality, and spend time re-running and editing. A Routine runs the same steps every time. The output quality floor goes up, and the effort goes to zero.
